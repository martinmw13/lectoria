# Frontend NCM types are generated from the backend OpenAPI schema

The frontend stops hand-mirroring the NCM contract. `frontend/src/api/types.ts`
re-exports types generated by `openapi-typescript` from the backend's
`/openapi.json` (extracted headless via `app.openapi()`, no running server). A CI
gate regenerates the schema and diffs the committed copy so any backend‚Üîfrontend
contract drift fails the build. Response DTOs use their backend names on the
frontend too (`SceneTrackResponse`, `DetailedSceneTrackResponse`, `MusicPreset`)
‚Äî no alias layer, one name per contract. To make the schema a complete source of
truth, `GET /books/{id}/chapters` is tightened from a `RootModel[dict]`
passthrough to `response_model=ChaptersData`, validated at the route boundary
while **BookStore stays data-faithful** (still returns raw JSON, no model
round-trip), preserving D15.

## Considered options

1. Types-only `openapi-typescript` + the existing hand-written client *(chosen)*
   vs a full client generator (`orval`/`openapi-fetch`) ‚Äî rejected to keep the
   lean `jsonFetch`/`processBook`/`providerHeaders` wrappers.
2. Commit generated artifacts + a CI diff gate *(chosen)* vs build-time
   generation ‚Äî chosen so `tsc` runs against committed types and drift is a
   reviewable diff.
3. Tighten `/chapters` *(chosen)* vs keep the `RootModel` passthrough as a
   documented exception ‚Äî rejected because the round-trip is lossless and the
   reader needs the structured type.

## Consequences

Route response validation now couples read-time health to `ChaptersData` schema
stability ‚Äî a future **required** field would 500 already-processed books.
Accepted because the current required fields (`Chapter.chapter_index`,
`Paragraph.index`, `Paragraph.text`) have been stable since the model was
introduced and the rest default (so only *optional* additive changes are safe),
and `data/books/` is regenerable. The route validates in-handler and logs with
`book_id`, so a corrupt artifact surfaces a traceable 500 rather than an
unattributed framework error.
