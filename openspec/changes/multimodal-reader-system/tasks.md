## Phase 2: Narrative Analysis Pipeline

- [x] 2.1 **LLM 1 service — book-level analysis** — Implement lectoria/services/narrative.py (LLM 1 portion): prompt template for character extraction (name, aliases, physical description, role, relationships), setting, genre, chapter summaries. Token count check against provider.max_context_tokens(). API call via LLMProvider. Validate response against BookMap schema. Retry logic (up to 3 attempts) for malformed JSON, fill defaults for missing optional fields.
- [x] 2.2 **LLM 2 service — scene-level analysis** — Implement lectoria/services/narrative.py (LLM 2 portion): prompt template with numbered paragraphs input + BookMap as context. Produces scenes with all attributes (emotion, pacing, scene_type, setting, image_prompt, transition_type, key_phrases, key_objects) + cover_description. Per-chapter processing. Validate against ChapterAnalysis schema. Scene coverage validation: no gaps, no overlaps; orphan paragraphs assigned to nearest scene. Retry logic.
- [x] 2.3 **NCM assembly + persistence** — Merge LLM 1 (BookMap) + all LLM 2 (ChapterAnalysis[]) outputs into a single NCM JSON. Post-processing: reconcile characters discovered by LLM 2 but missing from LLM 1. Generate book-id slug from title + author. Save ncm.json + chapters.json to data/books/<book-id>/.
- [x] 2.4 **Pipeline orchestration API** — POST /api/books endpoint: accept EPUB upload, store source.epub, run ingestion → LLM 1 → LLM 2 → NCM assembly. Token counting for cost estimation (return estimate before processing starts, require confirmation). SSE endpoint for progress reporting with running cost counter. Error handling for partial failures (retry chapter, fallback to single-scene).
- [x] 2.5 **LLM output coercion + dev metadata (Decision 18)** — Coercion layer mapping ~40 common LLM-invented enum values to valid taxonomy entries before Pydantic validation. Strengthened LLM 2 prompt with explicit enum constraints.

## Phase 3: Music

- [x] 3.1 **Music library setup** — Curate MTG-Jamendo subset (~200-500 tracks): define selection criteria (genres, mood tags, duration, quality, minimum 3 tracks per emotion category). Extract/classify tags, assign emotion_primary per track. Build one-hot tag vectors. Persist music index as JSON. Notebook for exploration/validation of the curated set.
- [x] 3.2 **Scene-to-track matching service** — Implement lectoria/services/music.py: hand-crafted mapping table (scene emotion/pacing/scene_type → Jamendo tag vocabulary). Two-phase matching: (1) filter by emotion_primary, (2) rank by cosine similarity of one-hot vectors. Variety rule: no same track in consecutive scenes. Hysteresis logic: emotion clusters (positive/dark/melancholic/neutral), short-scene threshold, crossfade decision per cluster distance. API endpoint: GET /api/books/<id>/scenes/<n>/track.

## Phase 4: Image Generation

- [x] 4.1 **Automatic image generation service** — Implement lectoria/services/image.py (automatic portion): for each scene, generate image from scene.image_prompt via ImageProvider. Generate chapter covers from cover_description. Save to data/books/<book-id>/images/scenes/ and images/covers/. Configurable: enabled/disabled per user settings. Integrated into offline pipeline (Phase 2.4) as optional final step.
- [x] 4.2 **On-demand image generation + character memory** — Implement lectoria/services/image.py (on-demand portion): receive selected text + scene context. Character identification: string matching against NCM character list (names + aliases) with possessive normalization ("'s" stripping). Fallback to scene.characters_present when no match. Inject physical descriptions. Character memory: store first generated image per character in images/characters/, retrieve and pass as reference if provider.supports_reference_image(). API endpoint: POST /api/books/<id>/images/generate.

- [x] 5b.1 **Unit test suite** — Tests for all deterministic service logic: `_coerce_scene_enums` and `_extract_json` (narrative.py), `identify_characters` and `build_on_demand_prompt` (image.py), `scene_to_vector`, `match_scene_to_track`, `should_crossfade`, and `assign_emotion_primary` (music.py), `ChapterAnalysis.validate_scene_coverage` (ncm.py model validator), `ingest_epub` with fixture EPUBs (ingestion.py). - [x] 5b.2 **Cache `get_settings()`** — Apply `functools.lru_cache` to `get_settings()` in `core/config.py` so env vars are parsed once per process, not per call.
- [x] 5b.3 **Fix triple `identify_characters` call** — In `image.py` `generate_on_demand`, call `identify_characters` once and reuse the result across prompt building, reference image selection, and character memory storage.

- [x] 5b.5 **Vectorize music matching** — Pre-stack track tag vectors into a numpy matrix at index load time. - [x] 5b.6 **Parallel chapter processing** — In `run_pipeline`, replace the sequential chapter loop with `asyncio.gather` bounded by a `Semaphore(N)` (configurable, default 3). Preserve progress reporting order. Handle partial failures (one chapter failing shouldn't abort others).

- [x] 5b.8 **Split route file** — Break `api/routes/books.py` into `books.py` (upload, list, get, process), `music.py` (track matching, crossfade), `images.py` (on-demand generation). Extract NCM chapter/scene lookups into a service helper to keep routes thin.
